# KZ Car Market — сбор и анализ авто-объявлений (kolesa.kz, Алматы)

Программа каждый день собирает объявления о продаже машин с kolesa.kz,
чистит данные, помечает подозрительные (приманки, перекупы, ворованные
фото) и оценивает справедливую цену. Итоговая цель — «умная оценка цены»:
сколько машина реально стоит и не обман ли это объявление.

## Что она умеет
- **Собирает** объявления (характеристики, фото, цена) и следит, живо ли объявление.
- **Помечает подозрительные**: слишком дёшево для такой машины; «одно фото — разные машины»; перезаливы.
- **Оценивает цену** моделью (CatBoost): даёшь характеристики → получаешь справедливую цену.
- **Показывает** всё в дашбордах — картинки и HTML-страница.

## Как пользоваться

### 1. Собрать данные (ходит в интернет)
```bash
python run_all.py
```
Полный цикл: собрать листинг → проверить статусы → обогатить страницы →
почистить → сделать отчёт. Другие режимы:
```bash
python run_all.py --light   # только новый листинг + пересборка (без тяжёлой сети)
python run_all.py --fast    # вообще без интернета: пересобрать отчёты из имеющегося
```
Добрать накопленные пробелы отдельно:
```bash
python catch_up.py --run              # по одной порции на джоб
python catch_up.py --run --backfill   # только avgPrice+бейдж у уже обогащённых
```
Программа сама тормозит, чтобы не словить бан от сайта (случайные паузы,
дневной лимит запросов на один сайт, стоп при сигнале «слишком часто»).
Если бан всё же случился — не долби, просто подожди день; всё
резюмируемо и продолжится с того же места.

### 2. Посмотреть результаты (интернет НЕ нужен)

| команда | что показывает | где смотреть |
|---|---|---|
| `python train_price_model.py` | строгая CV + baseline + обученный артефакт | `data/models/price_model.*` |
| `python residual_detector.py` | калибрует ценовой пол и строит кандидатов | `data/models/price_floor.*` |
| `python ml_report.py` | нарядный отчёт по модели цены | открой `data/eda/ml_report.html` в браузере |
| `python ml_dashboard.py` | графики модели | `data/eda/ml_dashboard.png` |
| `python explore.py` | обзор данных + подозрительные | `data/eda/dashboard.png` |
| `python predict_price.py` | оценка цены на примерах + разбор ошибки | в консоли |
| `python pipeline_status.py` | сколько собрано / сколько осталось | в консоли |
| `python evaluate_detector.py` | точность детектора (после разметки) | в консоли |

### 3. Узнать цену конкретной машины
Сначала один раз обучи и сохрани артефакт:
```bash
python train_price_model.py
```
Затем открой `predict_price.py`, найди вызов `estimate(...)` и впиши машину:
```python
estimate(m, brand="Toyota", model="Camry", year=2019, engine_volume=2.5,
         mileage_km=90000, engine_type="бензин", transmission="автомат",
         body_type="седан", condition="б/у")
```
Запусти `python predict_price.py` — он загрузит сохранённую модель, не будет
молча переобучать другую версию.

### 4. Разметить подозрительные (делает человек)
1. Открой `data/eda/labeling_queue.csv`. В ней три слоя:
   `rule_positive`, `residual_candidate` и `random_control`.
2. Для каждого поставь `verdict`: `fraud` (обман), `legit` (честно) или `unknown`.
3. Впиши вердикты в `data/manual_labels.csv` — **только дописывай, ничего не удаляй** (это ручной журнал).
4. Запусти `python evaluate_detector.py` — покажет, насколько детектор прав.

Не пропускай `random_control`: только случайная проверка невыделенных объявлений
может найти false negatives. Разметка одних подозрительных оценивает precision,
но не даёт честного recall.

## Как устроено (кратко)
```
parser.py        → собирает объявления              (raw_ads, sightings, photos)
check_status.py  → жив / продан / удалён            (ad_status)
clean.py         → чистит + ставит флаги            (clean_data)
enrich.py        → добирает данные со страницы       (растаможка, руль, привод, комментарий)
photo_dedup.py   → «одно фото — разные машины»       (photo_duplicates)
explore.py       → отчёт по данным
train_price_model / residual_detector / ml_report   → модель цены и антифрод
```
Данные лежат в Postgres (поднимается docker'ом). Сырьё (raw) не меняется —
только дополняется; чистые данные пересобираются из сырья при каждом запуске
(поэтому правки правил применяются задним числом).

Папки `data/` и `logs/` не хранятся в git — это данные, а не код; они
появляются при первом запуске.

## Модель цены — где сейчас
- **Grouped 5-fold CV:** MAPE 23.7%, R²(log)=0.915. Точные перезаливы
  удерживаются в одном фолде и не создают утечку.
- **Out-of-time holdout:** MAPE 22.9%, R²(log)=0.911 на 769 самых новых
  объявлениях.
- Простой baseline (медиана brand/model/year с fallback) даёт MAPE 34.5–37.6%;
  выигрыш CatBoost доказан относительно понятной отправной точки.
- Ошибка неоднородна: до 5 млн ₸ MAPE 31.3%, в сегментах 5–20 млн — около 19%.
- Учится только на «чистых» объявлениях (приманки/накрутки исключены,
  чтобы не отравляли модель).
- Не использует чужую оценку цены и производные от цены (чтобы не «списывать»).
- Модель, схема признаков, commit, метрики и fingerprint данных сохраняются
  вместе. Inference проверяет совместимость схемы.
- Точнее станет с бОльшим объёмом данных и с анализом фото.

Квантильный residual-детектор откалиброван out-of-fold: при целевом
`alpha=0.10` доля честных цен ниже пола равна 10%, а не прежним 20%.
Это только очередь кандидатов. Качество fraud-классификации пока нельзя
заявлять: валидного ground truth недостаточно.

## Тесты
```bash
python -m pytest test_pipeline.py -q
```
59 тестов. Проверяют логику на конкретных примерах, включая защиту ML-валидации
от дублей, хронологический split, калибровку пола и совпадение train/inference
схемы. GitHub Actions запускает их на каждом push и pull request.
